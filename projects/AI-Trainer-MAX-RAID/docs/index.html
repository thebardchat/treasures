Module shape

Every module. Same spine.

Predictable structure is how beginners survive. Four files per module. Four moves.

Lesson

Exercise

Verify
Automated pass/fail checks with specific failure reasons. No guessing.
04

File tree

One launcher. Everything else is modules.

Drop into the folder, run launch-training.bat, start with Module 1.1. The launcher handles health checks, progress tracking, and module navigation.

C:\AI-Trainer-MAX> tree /F
AI-Trainer-MAX/ ├── launch-training.bat # Main entry point — start here ├── config.json # Module registry + metadata ├── phases/ │ ├── phase-1-builders/ # 5 modules — Ollama, vectors, RAG │ ├── phase-2-operators/ # 7 modules — Business brain, workflows │ ├── phase-3-everyday/ # 7 modules — Vault, chat, briefings │ ├── phase-4-legacy/ # 7 modules — YourNameBrain, storytelling │ └── phase-5-multipliers/ # 10 modules — Hardening, teaching ├── progress/ │ └── user-progress.json # Auto-tracked completion data └── shared/ ├── ascii-art/ # CLI branding assets └── utils/ ├── health-check.bat # Ollama + Weaviate check ├── mcp-call.py # MCP client (stdlib only) └── mcp-health-check.bat # MCP server check
